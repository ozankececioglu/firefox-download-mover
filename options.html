Download Mover

One rule per line: <url-regex> => <target-path>. Use named capture groups (?<name>...) in the URL and reference them as {name} in the path. Numbered groups {1}, {2}… also work. Lines starting with # are comments.

Paths are relative to your Downloads folder. Prefixes ~/Downloads/, ~/ and leading / are stripped. Arbitrary filesystem paths are not supported (Firefox WebExtension limitation).

Verbose logs appear in the background console: about:debugging → This Firefox → Download Mover → Inspect.
